## 项目概述

开发一个医疗通用知识库rag问答web系统，能够处理用户输入的问题，并从医疗文档中检索相关信息以生成准确的回答。系统将结合基于文本的检索和基于向量的检索方法，以提高答案的相关性和准确性。同时，系统将提供一个用户友好的界面，允许用户方便地输入问题并查看答案。   

## 功能需求

1. **安全检测**：系统能够检测用户输入的问题是否包含敏感信息，并进行适当的处理以保护用户隐私。
2. **文档解析**：系统能够解析各种格式的医疗文档（如PDF、Word、markdown等），并提取其中的文本和表格内容以供后续处理。
3. **语义分片**：解析文档并将其分成适当大小的片段，以便更有效地进行检索和回答生成。
4. **混合检索**：系统能够同时使用基于文本的检索和基于向量的检索来获取相关信息。
5. **重排序**：系统能够对检索到的信息进行重排序，以提供最相关的答案。
6. **置信度评估**：系统能够评估答案的置信度，并在回答中提供相关信息的置信度评分。
7. **降级处理**：当系统无法提供准确答案时，能够提供相关信息的降级处理，以帮助用户更好地理解问题和答案的局限性。
8. **引用来源**：系统能够在回答中引用相关信息的来源，以增加答案的可信度和可验证性。
9. **风险提示**：系统能够在回答中提供相关信息的风险提示，以帮助用户更好地理解答案的局限性和潜在风险。
10. **多轮对话**：系统能够支持多轮对话，允许用户在一次会话中连续提问，并根据上下文提供相关的回答。
11. **用户界面**：系统提供一个美观清晰的web界面，允许用户方便地输入问题并查看答案。

## 核心技术栈

- **编程语言**：Python
- **包管理**：uv
- **后端框架**：FastAPI
- **前端框架**：Streamlit
- **RAG框架**：LlamaIndex
- **文档解析**：PyMuPDF、python-docx、markdown库
- **向量数据库**：Qdrant
- **大语言模型**：DeepSeek API
- **Embedding模型**：BAAI/bge-m3
- **Reranker模型**：BAAI/bge-reranker-v2-m3
- **SQL数据库**：PostgreSQL
- **缓存**：Redis
